03

Data Flow — A Filtered Request

How a filter selection becomes rendered data. The four filters apply across views.

User selects a filter

The FilterBar updates shared refs in useFilters — selectedPeriod, selectedLocation, selectedCategory, selectedStatus. State is a module-level singleton, so every view sees the same selection.

View requests data

A view calls getCurrentFilters() (maps period → month, location → warehouse) and passes the object to the matching api.js method.

axios builds the request

The client drops any value equal to 'all' and appends the rest as query params, then issues GET to http://localhost:8001/api/….

FastAPI filters in memory

The handler runs apply_filters() (warehouse / category / status, case-insensitive) and filter_by_month() (direct YYYY-MM match or quarter expansion) over the in-memory lists — no mutation of source data.

Pydantic validates the response

Results are returned through a response_model, enforcing the schema before serialization to JSON.

Vue renders via computed

Raw data lands in refs; derived metrics, charts (custom SVG), and tables are computed properties that recalculate only when their dependencies change.

05

Architectural Notes

Conventions and constraints worth knowing before making changes.

Singleton filter state

useFilters declares its refs at module scope, so all views share one filter selection — selecting a warehouse on Dashboard persists when navigating to Orders.

Refs in, computed out

Raw API responses live in refs (allOrders, inventoryItems); all derived values are computed so they cache and recalc only on dependency change.

Time dimension is order-only

Month/quarter filtering keys off order_date, so it applies to orders and the dashboard — inventory has no time dimension and ignores the month filter.

Stateless & ephemeral

Data is loaded once at startup and never written back. There is no auth or database; CORS is open. Suitable for local demo only.

Bilingual UI

useI18n drives English / Japanese with a key-path translator and localStorage persistence; locale also selects currency (en → USD, ja → JPY).

Charts are hand-built

No charting library — visualizations are custom inline SVG with CSS Grid layouts, computed from the reactive data.
